## Session Format

### 1. Opening (Patricia)
- State the documentation challenge or topic
- Set objectives for the session
- Identify which perspectives are most relevant

### 2. Initial Perspectives
Each relevant specialist shares their initial thoughts:
- What documentation is needed from their perspective
- Key considerations for their domain
- Potential challenges or dependencies

### 3. Discussion
- Cross-pollination of ideas between specialists
- Identify overlaps and gaps
- Resolve conflicting approaches
- Build on each other's suggestions

### 4. Synthesis (Patricia)
- Summarize key decisions and agreements
- Assign responsibilities for documentation tasks
- Define next steps and timeline

## How to Use This Skill

Invoke this skill when you need:
- Multiple perspectives on a documentation project
- Consensus on documentation strategy
- Collaborative review of existing documentation
- Planning for complex documentation efforts

## Example Topics

- "How should we document this new microservice?"
- "What's the best approach for our API versioning documentation?"
- "Review our onboarding documentation from multiple perspectives"
- "Plan documentation for a major feature release"

## Output

Sessions produce:
1. **Discussion Summary**: Key points from each specialist
2. **Decisions Made**: Agreed-upon approaches
3. **Action Items**: Specific documentation tasks with owners
4. **Timeline**: Proposed schedule for documentation work
