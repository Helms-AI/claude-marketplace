# Executive Summary Style

## Purpose
Use this style for stakeholder updates, status reports, and high-level decision briefings.

## Format Guidelines

### Structure
1. **Bottom Line Up Front (BLUF)**
   - Key decision/finding in first sentence
   - Impact in second sentence
2. **Status** (use icons)
   - ✅ On track
   - ⚠️ At risk
   - 🔴 Blocked
3. **Key Points** (3-5 bullets max)
4. **Action Required** (if any)
5. **Timeline**

### Example Format
```
## Project Alpha - Weekly Status

**Status: ⚠️ At Risk**

Authentication migration 80% complete but blocked on security review.

### Key Points
- User migration: 45,000/50,000 accounts migrated (90%)
- Performance: 23% improvement in login latency
- Blocker: Security team review scheduled for Thursday

### Action Required
- [ ] @security-lead: Expedite auth review
- [ ] @pm: Update stakeholder timeline

### Timeline
- Security review: Thu Mar 15
- Go-live: Mon Mar 19 (was Mar 18)
```

### Tone
- Concise and action-oriented
- Lead with impact, not process
- Numbers over adjectives
- Clear ownership on action items

### Length
- Maximum 300 words
- Use bullets over paragraphs
- One page or less when printed
