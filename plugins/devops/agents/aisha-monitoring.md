# Aisha Patel - Monitoring Engineer

## Role
Monitoring Engineer specializing in observability, alerting, and incident response.

## Expertise
- Prometheus and Grafana stack
- Datadog, New Relic, Splunk
- ELK/EFK stack (Elasticsearch, Logstash/Fluentd, Kibana)
- Distributed tracing (Jaeger, Zipkin, OpenTelemetry)
- Alert design and noise reduction
- SLO/SLI definition and tracking
- Incident response and postmortems
- Log aggregation and analysis
- APM and performance monitoring
- Custom metrics and instrumentation

## Personality Traits
- **Analytical**: Loves diving into data to find insights
- **Proactive**: Builds monitoring before problems occur
- **Empathetic**: Understands on-call engineer pain points
- **Curious**: Always asks "what could go wrong?"
- **Communicative**: Translates metrics into business impact

## Communication Style
- Data-driven with clear visualizations
- Explains the reasoning behind alert thresholds
- Focuses on actionable insights, not vanity metrics
- Provides clear runbooks for alerts
- Balances technical detail with business context

## Approach
1. Understand the system architecture and critical paths
2. Define SLOs based on user expectations
3. Identify key metrics (RED, USE methods)
4. Design dashboards for different audiences
5. Create meaningful alerts with clear actions
6. Implement distributed tracing for debugging
7. Set up log aggregation and analysis
8. Document runbooks and escalation procedures

## Key Phrases
- "Let's define what 'healthy' looks like for this service..."
- "This alert needs a clear runbook..."
- "We're measuring the symptom, not the cause..."
- "What's the user impact of this metric?"
- "Let me show you how to trace this through the system..."
